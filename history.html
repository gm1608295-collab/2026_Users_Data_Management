🛡️ လုံခြုံရေး အလွှာ (၄) ခု

🔒 Layer 1 - Security Password

သင့် Data တွေကို ကြည့်ဖို့ Security Password ထည့်ရပါတယ်။ Password မထည့်ရင် ဘယ်သူမှ မြင်ရမှာ မဟုတ်ပါ။

🔐 Layer 2 - JWT Token စစ်ဆေးခြင်း

API တိုင်းမှာ JWT Token ကို စစ်ဆေးပါတယ်။ သင့် Token မရှိရင် Data ကို ဝင်ကြည့်လို့ မရပါ။

👤 Layer 3 - User Isolation

သင့် Data ကို သင်ပဲ ကြည့်လို့ရပါတယ်။ တခြားသူရဲ့ Data ကို မြင်ရမှာ မဟုတ်သလို သင့် Data ကိုလည်း တခြားသူ မြင်ရမှာ မဟုတ်ပါ။

⏰ Layer 4 - Attempt Limit & Cooldown

Password ၅ ကြိမ်မှားရင် ၁၀ မိနစ် Lock ကျပါတယ်။ Brute Force Attack ကို ကာကွယ်ထားပါတယ်။

🛡️ SOLO M Game Shop သည် သင့်ဒေတာလုံခြုံရေးကို အမြင့်ဆုံးဦးစားပေးပါသည်။

⚠️ သတိပြုရန် - မကာကွယ်နိုင်သော အခြေအနေများ

🗣️ ကိုယ်တိုင် ပြောပြခြင်း / မျှဝေခြင်း

သင့် Password သို့မဟုတ် Data များကို သင့်ဘာသာ တခြားသူကို ပြောပြခြင်း၊ Screenshot ရိုက်ပို့ခြင်း၊ Screen Recording လုပ်ပို့ခြင်းတို့ကို စနစ်က ကာကွယ်နိုင်မည် မဟုတ်ပါ။

📱 Device ကိုယ်တိုင် လုံခြုံမှုမရှိခြင်း

သင့်ဖုန်းကို Lock မချထားခြင်း၊ တခြားသူကို ဖုန်းငှားပေးထားခြင်း၊ အများသုံး Device တွင် Login ဝင်ထားခြင်းတို့ကို စနစ်က ကာကွယ်နိုင်မည် မဟုတ်ပါ။

🔓 Password အားနည်းခြင်း

ခန့်မှန်းရလွယ်သော Password (ဥပမာ - 123456, password, မွေးနေ့) တို့ကို သုံးထားပါက စနစ်က ကာကွယ်နိုင်မည် မဟုတ်ပါ။

🎣 Phishing / လိမ်လည်ခံရခြင်း

အတုအယောင် Website များ၊ လိမ်လည်သူများကို သင့် Password ကို ကိုယ်တိုင်ထည့်ပေးမိပါက စနစ်က ကာကွယ်နိုင်မည် မဟုတ်ပါ။

💾 Data Backup မရှိခြင်း

သင့် Data များကို စနစ်ထဲတွင် သိမ်းဆည်းထားသော်လည်း သင့်ဘာသာ Backup မလုပ်ထားပါက စနစ်ချို့ယွင်းမှုကြောင့် Data ဆုံးရှုံးနိုင်ခြေကို လုံးဝ အာမခံနိုင်မည် မဟုတ်ပါ။

📊 ကာကွယ်နိုင်သည် vs မကာကွယ်နိုင်ပါ

✅ ကာကွယ်နိုင်သည် ❌ မကာကွယ်နိုင်ပါ
Hacker တိုက်ခိုက်မှု ကိုယ်တိုင် Password ပြောပြမိခြင်း
Database ပေါက်ကြားမှု ဖုန်း Lock မချထားခြင်း
Brute Force Attack အများသုံး Device တွင် Login ဝင်ခြင်း
တခြားသူ Data ကြည့်ခြင်း Phishing Site တွင် Password ထည့်မိခြင်း
Session ခိုးယူမှု Screenshot/Recording ရိုက်ပို့ခြင်း
Password အလွဲသုံးစားမှု Weak Password သုံးထားခြင်း
